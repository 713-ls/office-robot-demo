# 办公环境自主移动机器人项目开发文档

## 1. 项目定位

本项目面向简历作品集展示，目标是在仿真环境中实现一个办公环境自主移动机器人 demo。机器人需要完成室内自主导航、语义目标识别、任务触发、简化拾取与配送，并通过 RViz 和录屏材料展示完整流程。

项目首版只做仿真闭环，不依赖真实机器人硬件。抓取部分采用简化策略：机器人到达目标附近后，通过任务状态和可视化标记模拟“拾取”和“放置”，不实现机械臂运动学和真实夹爪控制。

## 2. 项目目标

### 2.1 最小可展示目标

- 在 Gazebo 中搭建办公场景，包含办公桌、通道、障碍物和目标物体。
- 使用 TurtleBot3 Waffle Pi 在地图中完成多目标点导航。
- 使用 ROS Navigation Stack 完成路径规划、局部避障和返回起点。
- 使用仿真相机识别办公物体，例如瓶子、文件夹、箱子。
- 将识别结果发布为 ROS topic，并触发任务管理节点。
- 完成“发现目标 -> 前往目标 -> 简化拾取 -> 送达指定区域 -> 返回起点”的完整流程。
- 使用 RViz 展示路径、机器人状态、目标检测结果和任务状态。
- 使用 Git 记录阶段性成果，并整理 README、截图、演示视频。

### 2.2 作品集展示目标

- 让面试官能快速看懂项目做了什么、技术栈是什么、你负责了哪些模块。
- 让项目仓库具备清晰结构、可运行命令、架构图、阶段成果和演示材料。
- 用提交记录体现工程推进过程，而不是一次性堆代码。

## 4. 环境搭建

### 4.1 基础环境

推荐在 Ubuntu 20.04 中开发。Windows 用户建议使用 Ubuntu 双系统、虚拟机或 WSL2 图形环境。由于当前本机 WSL 暂无可用 Linux 发行版，第一阶段需要优先完成 Linux/ROS 环境准备。

基础安装清单：

```bash
sudo apt update
sudo apt install -y git python3-pip python3-catkin-tools
sudo apt install -y ros-noetic-desktop-full
sudo apt install -y ros-noetic-navigation ros-noetic-map-server ros-noetic-amcl
sudo apt install -y ros-noetic-turtlebot3 ros-noetic-turtlebot3-simulations
sudo apt install -y ros-noetic-cv-bridge ros-noetic-image-transport
```

ROS 环境变量：

```bash
echo "source /opt/ros/noetic/setup.bash" >> ~/.bashrc
echo "export TURTLEBOT3_MODEL=waffle_pi" >> ~/.bashrc
source ~/.bashrc
```

### 4.2 工作空间

建议工作空间结构：

```text
catkin_ws/
  src/
    office_robot_sim/
    office_robot_navigation/
    office_robot_perception/
    office_robot_task/
    office_robot_visualization/
```

创建工作空间：

```bash
mkdir -p ~/catkin_ws/src
cd ~/catkin_ws
catkin_make
echo "source ~/catkin_ws/devel/setup.bash" >> ~/.bashrc
source ~/.bashrc
```

## 6. 开发规范

### 6.1 Git 分支

推荐分支策略：

- `main`：稳定版本，保存可展示 demo。
- `dev`：日常集成分支。
- `feature/simulation`：Gazebo 场景和机器人模型。
- `feature/navigation`：导航、地图、目标点、回起点。
- `feature/perception`：目标检测与识别结果发布。
- `feature/task-flow`：任务编排、简化抓取、配送流程。
- `docs/project-docs`：项目文档和作品集材料。

提交建议：

```text
docs: add project architecture and roadmap
sim: add office world and turtlebot launch
nav: add multi-goal navigation node
perception: publish detected object topic
task: add pick-and-deliver state machine
```

### 6.2 代码规范

- Python 节点使用清晰的 ROS node 名称和 topic 名称。
- 每个 ROS package 保留独立 `README.md`，说明启动命令和依赖。
- 配置文件放入 `config/`，launch 文件放入 `launch/`，地图放入 `maps/`，模型放入 `models/`。
- 任务状态统一枚举，例如 `IDLE`、`DETECTED`、`NAVIGATING_TO_OBJECT`、`PICKING`、`DELIVERING`、`DONE`、`FAILED`。

## 7. 阶段开发内容

### 7.1 阶段一：环境搭建

目标：ROS Noetic、Gazebo、RViz、TurtleBot3 仿真能正常启动。

任务：

- 安装 Ubuntu 20.04 和 ROS Noetic。
- 安装 TurtleBot3 仿真包。
- 创建 catkin 工作空间。
- 跑通 TurtleBot3 Gazebo demo。

验收：

- Gazebo 中能看到 TurtleBot3。
- RViz 中能看到机器人模型、TF 和传感器数据。
- Git 完成第一次提交。

### 7.2 阶段二：Gazebo 办公场景

目标：搭建办公环境仿真地图。

任务：

- 创建办公室 world，包含通道、桌椅、障碍物、目标物体。
- 确保机器人可以在场景内通行。
- 放置至少 3 个导航目标点和 2 类可识别物体。

验收：

- Gazebo 场景能稳定启动。
- 机器人不会出生在障碍物中。
- 场景截图加入 README。

### 7.3 阶段三：自主导航

目标：机器人能在办公环境内自主移动、避障、到达目标点并返回起点。

任务：

- 使用 `gmapping` 或预建地图生成办公地图。
- 配置 `map_server`、`amcl`、`move_base`。
- 编写 `navigation_manager_node`，支持多目标点导航和回起点。

验收：

- 完成至少 3 个目标点导航。
- 能绕开静态障碍物。
- RViz 展示全局路径、局部路径和机器人位姿。

### 7.4 阶段四：语义感知

目标：机器人能通过仿真相机识别办公物体。

任务：

- 第一版使用 OpenCV 或轻量规则检测跑通流程。
- 第二版接入 YOLOv8n，识别瓶子、文件夹、箱子等目标。
- 将识别结果发布到 `/detected_objects`。

验收：

- 能识别至少 2 类目标。
- 输出类别、置信度和图像坐标。
- RViz 或调试窗口能展示检测结果。

### 7.5 阶段五：任务执行与简化抓取

目标：完成完整任务闭环。

任务：

- 编写 `task_manager_node`，接收目标检测结果。
- 根据目标类别选择取货点和送达点。
- 调用导航模块前往目标附近。
- 使用状态切换和 Marker 模拟拾取、携带和放置。
- 任务完成后返回起点。

验收：

- 任务状态完整可见。
- demo 能从目标识别自动走到配送完成。
- 失败时能进入 `FAILED` 状态并输出原因。

### 7.6 阶段六：作品集整理

目标：把项目整理成可展示作品。

任务：

- 完善 README。
- 添加架构图、运行命令、模块说明。
- 添加 Gazebo、RViz、目标检测、完整 demo 的截图。
- 录制 1 到 3 分钟演示视频。
- 整理关键 Git 提交记录。

验收：

- 面试官打开仓库能在 3 分钟内理解项目亮点。
- README 能独立说明项目如何运行和展示。

## 8. 风险与对策

| 风险 | 影响 | 对策 |
| --- | --- | --- |
| ROS 环境搭建失败 | 无法开发 | 优先完成 Ubuntu 20.04 + ROS Noetic 安装验证 |
| Gazebo 运行卡顿 | demo 不稳定 | 场景先保持简单，减少模型数量 |
| YOLO 训练数据不足 | 识别效果差 | 先用预训练模型或 OpenCV 规则跑通流程 |
| 导航调参耗时 | 路径规划不稳定 | 先使用 TurtleBot3 默认参数，再逐步调整 |
| 抓取实现复杂 | 影响项目完成 | 首版只做简化抓取状态机 |
