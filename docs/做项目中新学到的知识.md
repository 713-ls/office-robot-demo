# 做项目中新学到的知识

## 1.什么是.world文件
Gazebo 的世界文件是 SDF 格式,本质是一个 XML 文件。它告诉 Gazebo：地面长什么样、光照怎么打、物理参数是什么、有哪些模型、各自摆在什么位置。

和 URDF 的区别：URDF 描述单个机器人，SDF 描述整个场景 + 场景里的一切。

<physics> — 物理引擎设置，有物理仿真步长，仿真速度，频率等
<light> — 光照
<model> — 物体
<visual name="visual">，给渲染用，决定物体长什么样
<collision name="collision">，给物理引擎用，决定碰撞形状
<link name="link">，把俩个包在一起
<static>，物体固定不动
<pose>，位置和朝向
<material>，颜色
<ambient>,环境光颜色
<diffuse>，漫反射颜色

写法：<box><size>1.2 0.6 0.03</size></box>类似这种把值包住

参数：
| x | 前后位置 | 米 |
| y | 左右位置 | 米 |
| z | 上下位置 | 米 |
| roll | 绕x轴旋转（翻滚） | 弧度 |
| pitch | 绕y轴旋转（俯仰） | 弧度 |
| yaw | 绕z轴旋转（偏航） | 弧度 |

export TURTLEBOT3_MODEL=waffle_pi
roslaunch turtlebot3_gazebo turtlebot3_empty_world.launch

Gazebo 左侧面板 → World 标签 → 展开 Models → 右键 turtlebot3_waffle_pi → Delete。

按 Ctrl + B按规划好的布局放好墙壁，放好后先保存好后退到gazebo原界面再去insert放其他模型。建议先放到场景后再去按坐标调整它的pose。

第 5 步：保存世界文件
Gazebo 顶栏 → File → Save World As → 保存到 ~/office_robot_ws/src/office_robot_sim/worlds/office.world。

第 6 步：查看保存的 .world 文件
cat ~/office_robot_ws/src/office_robot_sim/worlds/office.world

第 7 步：验证场景能否独立启动
gazebo ~/office_robot_ws/src/office_robot_sim/worlds/office.world
用ros下启动：
rosrun gazebo_ros gazebo ~/office_robot_ws/src/office_robot_sim/worlds/office.world
第 8 步：把 TurtleBot3 Waffle Pi 加回场景
终端 1

rosrun gazebo_ros gazebo ~/office_robot_ws/src/office_robot_sim/worlds/office.world

## 4.解决slam建图时机器人穿过过高的物体问题
### 问题描述
在自定义 Gazebo 办公室世界中进行 SLAM 建图和自主导航时，TurtleBot3 waffle_pi 会从桌子下方穿过。原因是桌面高度较高，而 2D 激光雷达扫描高度较低，雷达只能看到桌腿，无法把整张桌子识别为完整障碍物，导致导航代价地图把桌子下方区域判断为可通行区域。
### 问题原因
Gazebo 中桌子的视觉模型正常显示，但其可被导航感知到的障碍信息不足。对于 TurtleBot3 waffle_pi 来说，2D 雷达主要用于 SLAM 和避障，如果桌子主体高于雷达扫描平面，机器人就可能在路径规划时从桌面下方穿过。
### 解决思路
不修改原有桌子模型的外观和位置，只在每张桌子下方添加不可见的碰撞体，使导航系统把桌子占据区域识别为障碍物。
### 解决过程
检查 .world 文件中的桌子模型，确认桌子由桌面和桌腿组成，桌面高度较高。
初步尝试给桌子添加低位薄碰撞板，但机器人仍可能从桌子下方穿过。
将薄碰撞板改为更稳妥的不可见实心碰撞盒，覆盖桌子桌面投影范围。
给每张桌子添加如下碰撞体：
<collision name='navigation_laser_blocker'>
  <pose>0 0 0.48 0 -0 0</pose>
  <geometry>
    <box>
      <size>1.55 0.85 0.92</size>
    </box>
  </geometry>
  <surface>
    <contact>
      <ode/>
    </contact>
  </surface>
  <max_contacts>10</max_contacts>
</collision>

### 补充说明
该方法只增加 collision，不增加 visual，因此不会改变桌子的显示效果，也不会影响后续相机对桌面目标物体的视觉识别。

## 5.导航参数调整
### 复制TurtleBot3官方导航参数
cp /opt/ros/noetic/share/turtlebot3_navigation/param/*.yaml ~/office_robot_ws/src/office_robot_navigation/params/

### 修改launch地图指向
把全局变量map_file的值改为自己的地图路径，如：
<arg name="map_file" default="$(find office_robot_navigation)/maps/office_map.yaml"/>

### 把世界加入导航launch文件里
把原launch里关于Turtlebot3：
<!-- Turtlebot3 -->
<include file="$(find turtlebot3_bringup)/launch/turtlebot3_remote.launch">
  <arg name="model" value="$(arg model)" />
</include>
替换成：
<!-- Gazebo world -->
<include file="$(find gazebo_ros)/launch/empty_world.launch">
  <arg name="world_name" value="$(find office_robot_sim)/worlds/office.world"/>
  <arg name="paused" value="false"/>
  <arg name="use_sim_time" value="true"/>
  <arg name="gui" value="true"/>
  <arg name="headless" value="false"/>
  <arg name="debug" value="false"/>
</include>

同时最好加个全局时间：<param name="/use_sim_time" value="true"/>
/ 开头的斜杠表示全局参数。

### 编译：
cd ~/office_robot_ws && catkin_make && source devel/setup.bash

## 6.解决机器人模型在RViz里看不到的问题
### 原因：
world 里自带的机器人是静态壳，没有 TF、没有电机控制器、没有 sensor 插件。导航栈收不到 /tf、/scan，RViz里就看不到机器人。
### 解决：
把机器人从 world 文件里删掉，改由 launch 文件动态生成——动态生成才会带控制器、LIDAR 插件、相机插件和 TF。

在world文件里删掉机器人模型：
搜索turtlebot3_waffle_pi，删掉整个<model name="turtlebot3_waffle_pi">...</model>块。

在launch文件里加：
<param name="robot_description" command="$(find xacro)/xacro '$(find turtlebot3_description)/urdf/turtlebot3_waffle_pi.urdf.xacro'"/>
  <node pkg="gazebo_ros" type="spawn_model" name="spawn_urdf" args="-urdf -model turtlebot3_waffle_pi -x -3 -y -4 -z 0 -param robot_description"/>
  <node pkg="robot_state_publisher" type="robot_state_publisher" name="robot_state_publisher">
  <param name="publish_frequency" type="double" value="50.0"/>
</node>

## 7.解决机器人在rviz里没有空降到特定位置的问题
### 原因
这是AMCL的正常行为AMCL，刚启动时不知道自己在哪，默认把机器人放在地图原点 (0,0)。
### 解决
第一步：在rviz里用2D Pose Estimate手动给一个初始位姿估计。
第二步：在 navigation.launch 里传参，让 AMCL 启动时就用你指定的初始位姿。
找到有关AMCL的，加入位置变量，并以</include>结尾，同时记得把第一行的<include .../> ，中的/删去，不然会说标签不匹配：
<arg name="initial_pose_x" value="-3.0"/>
<arg name="initial_pose_y" value="-4.0"/>
<arg name="initial_pose_a" value="0.0"/>

## 9.actionlip
是"发任务 → 等结果"的通信机制,如告诉 move_base"我要去 (x,y)"，它自己规划、避障、走过去，走完后通知你"到了"或"死了"。你不用盯着它。
